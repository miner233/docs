Mkdocs 使用笔记

一切的开始：安装与配置

安装

今天是 2022 年 10 月 13 日，我终于成功配置了 Mkdocs。
今天我从上午就开始尝试，经过了数不清多少次的测试，终于成功了。
事实上，我早就开始尝试使用 Mkdocs 搭建文档了，至少在暑假的时候。
为了避免像我一样的新手踩雷，又是怕我忘记 Mkdocs 的使用方法，我决定进行记录。

第一次尝试

刚开始，我还很天真，想着就是装个 Mkdocs，pip 不就行了吗？于是，我开始了第一次尝试。
我当时用的版本是 Python 3.10.8，我看 这个 据说是从 Mkdocs 官方网站上面翻译的教程，我想：可是官方网站啊，官方网站肯定是最详细、最全面、最正确的文档了吧？
可能是没有更新的缘故，那个网站上面说 Mkdocs 最高支持版本是 Python 3.4，我立刻去官网下载了 Python 3.4.4，勾选“添加到 PATH”，我之前没有添加到环境变量的教训我可记忆犹新！安装完成后，我打开 cmd，输入：

pip install mkdocs

我正等待着，想着也是 3.4，所以不可能有问题的。结束以后，它并没有显示"error"之类的单词。
可以下一步了！我想，按照那个网站上的教程，输入 mkdocs help 测试是否安装完成。但是，cmd 中却显示了：

'mkdocs' 不是内部或外部命令，也不是可运行的程序或批处理文件。

后来，我又用其他的方法试了很多次，不管怎么做，都没有安装成功。

第 N 次尝试

网上冲浪发现别人用的 Python 版本是 Python 3.7。我立刻安装，下载安装包以后，我发现原来已经装好了。我又打开 cmd，运行了python --version。cmd 上显示的是Python 3.10.8。我又打开高级系统设置，立刻改了环境变量，然后又试了一遍。而这次，是Python 3.7.9！我高兴极了，知道已经成功了一半了。
再次在 cmd 中运行 pip，但上面却说未安装 pip。我又打开安装包，发现安装包上面说我已经安装了 pip。我只好先把 pip 取消安装，移除完以后重新安装。终于，这次找到了 pip！
这意味着我已经成功了 90%。我按网上的资料，资料上面说在 Windows 环境下只能使用python -m pip install mkdocs安装 Mkdocs。我按照上面的做。完成后，我试了试mkdocs help，我惊奇的发现，能够成功运行，终于安装完了！

配置

创建项目

这时，我进入了一个新的目录，运行了mkdocs new docs，然后又运行了mkdocs serve启动了本地预览，又用浏览器访问了 127.0.0.1:8000。成功了！这时，已经完成了 99%，下面就是更改文档内容了。

配置文档

运行mkdocs new docs会在目录下创建 mkdocs.yml 文件和 docs 文件夹。其中，mkdocs.yml 是文档基础配置。下面展示了该配置文件中的一些基础设置项目。

site_name: (站点名称)
site_description: (站点注释)
site_author: (站点作者名称)
site_url: (站点地址)

copyright: (版权声明相关文本)

# nav 区域指定站点上能看到的页面。
# 下面的配置代码仅作示范。注意配置的页面对应的 MarkDown 源码文件必须存在。
nav:
  - Index: index.md
  - Page 1: page1.md
  # 下面代表的是一组页面
  - Page Group:
    - Sub Page 1: subpage/page1.md
    - Sub Page 2: subpage/page2.md
  -About: about.md

下面就是编写文档了！Mkdocs 使用 Markdown 语法，这意味着你可以快速的编写你的文档。
